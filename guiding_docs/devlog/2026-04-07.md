## Changes

### PLAN.md revision 2 тАФ wiki split into its own repo
- Yesterday's draft put the wiki at `vault/wiki/` as a subtree of each per-agent vault. The user pushed back: that conflates two different concerns (per-agent private state vs. cross-agent shared knowledge), creates duplication, and gives the memory service a leaky boundary into per-agent vault mounts.
- Reframed: `commandclaw-wiki` is a new top-level repo, separate from `commandclaw-vault`. The memory service owns its working clone and never touches per-agent vaults. Agents reach the wiki only through the REST API of `commandclaw-memory`.
- Physical isolation replaces the previous `file_write` path guard тАФ the wiki simply isn't mounted into agent containers, so there's nothing to guard against.
- Added section 4.1a covering wiki clone lifecycle: clone on boot, sync commit + async push every `PUSH_INTERVAL` seconds, conflict handling (merge commit on push reject, 503 on merge conflict), single-writer constraint.
- Bootstrap section split into two operations: (a) provisioning the wiki repo manually one-time, (b) optional seed-from-vaults ingest via `POST /admin/bootstrap`.
- docker-compose volume mapping now mounts the wiki clone at `/wiki` instead of mounting the workspaces dir (which was a leaky boundary anyway).
- Build phasing grew from 11 to 12 phases. New Phase 1 creates the `commandclaw-wiki` repo. All later phases bumped by one.
- Open questions grew from 12 to 17, covering hosting, push cadence, divergence recovery, multi-tenancy.

### PLAN.md revision 3 тАФ wiki reading and browsing
- User asked: "is there a Wikipedia-like wiki viewer?" тЖТ triggered an entirely new section.
- Surveyed the space: BookStack, Wiki.js, Outline, Docmost (server-side), MkDocs Material, Docusaurus, Quartz, Astro Starlight (static site), Obsidian, Logseq, Foam (desktop reader).
- Picked **three viewers for three audiences**:
  - **Operator / debug** тЖТ Obsidian against the wiki clone on disk. Zero config. Graph view, backlinks, search out of the box. The repo ships with a committed `.obsidian/` config (Git plugin disabled to prevent accidental edits).
  - **Stakeholder / read-only** тЖТ Quartz static site, built by GitHub Action on every push to `main`. Quartz is the right pick because it handles frontmatter + wikilinks + auto-discovery natively, unlike MkDocs Material which expects curated nav.
  - **CLI / scripts** тЖТ REST API directly, plus a small `cclaw wiki` CLI wrapper as a Phase 9 deliverable.
- Reader stack adds **zero** ops surface to the memory service. Everything is consumer-side of the wiki repo. The memory service never serves HTML.
- Phase 1 expanded to ship `.obsidian/` + `quartz/` + `.github/workflows/publish.yml` alongside the seed content. Two new open questions about publish hosting (public / org-restricted / Cloudflare Access) and theme polish. Question count is now 19. Section count is now 15.

## Notes / observations
- The "split the wiki" conversation crystallized when the user said "we are designing it for enterprises and we will do it right." The previous v1/v2 framing тАФ Karpathy now, BEARY later тАФ was a bad shortcut. They are orthogonal layers, not competing alternatives. Karpathy's pattern is the data model and workflow contract; BEARY's service is the discipline layer that makes it work for cheap models. Both have to ship together. The corrected synthesis is what landed in PLAN.md rev 2 and what the new repos are scaffolded for.
- The reader stack section was a surprise scope addition тАФ the user asked an offhand question about Wikipedia-like viewers and that turned into a whole new section of the plan. It's the right addition though. A wiki with no viewer is just a directory.
- Quartz vs MkDocs Material was the most interesting reader choice. MkDocs is more polished out of the box but expects curated navigation in `mkdocs.yml`. Our wiki is dynamic тАФ pages come and go via the service тАФ so curated nav is the wrong shape. Quartz auto-discovers everything and handles wikilinks natively. Right tool for the right shape.
- Three commits to `commandclaw` today (`5bf9e2a`, `702bf8f`), two to new repos, and the workspace gained two new cloned dirs. PLAN.md is now 1100+ lines and 15 sections. Worth a re-read in a fresh session before signoff.
